# Curriculum Design Reference

## Learning Design Principles

**Design for completion, not consumption.** Students don't finish courses they don't understand or can't act on. Structure each lesson so there's a clear action at the end.

**The transformation framework:**
1. Start state: Where is the student today? (pain, gap, frustration)
2. End state: Where will they be after completing? (outcome, transformation)
3. Journey: The milestones between start and end — these become your modules

## Course Structure Templates

### Standard Multi-Module Course
```
Module 1: Foundation / Quick Win         (build confidence, validate decision)
Module 2: Core Concept A                 (moves student toward outcome)
Module 3: Core Concept B
Module 4: Core Concept C
Module 5: Putting It Together            (integration + implementation)
Module 6: Advanced / Next Level          (optional, for premium tiers)
Bonus:    Templates, Tools, Community    (add value without adding overwhelm)
```

### Mini-Course (1–3 hours)
```
Intro: Problem + Promise (5 min)
Lesson 1: Concept (10 min)
Lesson 2: Demonstration (10 min)
Lesson 3: Application (10 min)
Outro: Action + Next Steps (5 min)
```

### Cohort-Based Course (4–8 weeks)
```
Week 1: Orientation + Foundation
Week 2–6: Weekly skill-building modules
Week 7: Capstone project
Week 8: Showcase + community celebration
```

## Module Design Rules

- **One module = one concept or outcome.** If a module needs two explanations in the title ("X and Y"), split it.
- **Start with the "why."** Why does this module matter? What will they be able to do after?
- **End with an action.** Every module should close with something the student can do right now.
- **Module length:** 20–60 minutes of content max. Shorter is better for completion rates.

## Lesson Design Rules

- **5–15 minutes per lesson.** Long lessons reduce completion. Break anything over 15 min.
- **One lesson = one idea.** No "kitchen sink" lessons that cover 5 things.
- **Lesson naming:** Use outcome-focused names, not topic names.
  - Bad: "Introduction to Email Marketing"
  - Good: "Write Your First Welcome Email in 20 Minutes"

## First Module: The Critical Hook

The first module does three things:
1. **Validates the purchase** — confirms they made the right decision
2. **Delivers a quick win** — something they can implement or understand immediately
3. **Sets expectations** — what the journey looks like, how to get the most out of it

Students who complete a task within the first hour of purchase are 3x more likely to finish. Invest in a strong first module.

## Curriculum Outline Process

1. **Brain dump** — list everything you know about the topic, no filtering
2. **Group by theme** — cluster related items into potential modules
3. **Order by journey** — arrange modules so each one builds on the last
4. **Identify gaps** — what's missing to move from start to end state?
5. **Trim ruthlessly** — cut anything that doesn't directly serve the transformation
6. **Add quick wins** — identify 1–2 moments in the first module where students can get an immediate win

## Supporting Materials That Add Value

- **Worksheets** — students fill in as they go (e.g., a template to complete)
- **Checklists** — pre-flight checks, action lists
- **Templates** — ready-to-use assets (email templates, spreadsheets, frameworks)
- **Resource lists** — curated links, tool recommendations, reading lists
- **Assignments** — tasks to complete between lessons or modules
- **Quizzes** — knowledge checks (also improve retention and perceived value)

## Common Curriculum Mistakes

| Mistake | Fix |
|---------|-----|
| Too much content | Cut to the core journey; bonus material can go in an "advanced" module |
| No clear outcome per module | Add a "by the end of this module you will..." header |
| Weak first module | Add a quick win in the first 10 minutes |
| No action items | End every lesson with "your next step is..." |
| All video, no variation | Mix formats (video, text, worksheet, quiz) |
