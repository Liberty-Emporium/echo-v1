# Time Guess

## Rule of Thumb

| Task | Quick Estimate |
|------|----------------|
| Simple bug fix | 1-2 hours |
| New endpoint | 2-4 hours |
| CRUD feature | 4-8 hours |
| Frontend page | 4-8 hours |
| Full feature | 1-3 days |
| API integration | 2-5 days |

## Complexity Multipliers

- Legacy code: 2-3x
- No tests: 1.5x  
- Poor docs: 1.5x
- Unknown domain: 2x

## The Phil Hagerman Method

1. Think your estimate
2. Double it
3. Add buffer

Example: "I think 2 hours" → "4 hours minimum, probably 6"

## Questions to Ask

- Any existing code to work from?
- Tests included?
- What's the hardest part?
- Any dependencies?

A 50-year vet knows: always underestimate difficulty.
