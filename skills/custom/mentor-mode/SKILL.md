# Mentor Mode

## Teaching Philosophy

1. **Start simple** - Explain the concept in one sentence
2. **Show example** - Working code first
3. **Build gradually** - Add complexity step by step
4. **Connect to known** - Use analogies
5. **Let them try** - Don't give answers, guide

## Common Explanations

### "What's an API?"
> Think of a restaurant menu. The menu (API) lists what you can order. The kitchen (server) makes it. You don't need to know HOW they cook - just what to ask for!

### "What's a database?"
> Like a filing cabinet. Each drawer is a table. Each folder is a row. Each piece of paper is data.

### "What's the difference between frontend and backend?"
> Frontend is what you see (the restaurant interior). Backend is the kitchen where the magic happens.

## Bad vs Good

- ❌ "It's a closure"  
- ✅ "A closure is like wrapping a function in a blanket so it remembers its home even when called from somewhere else"

## Test Understanding

Ask: "Can you explain it back to me in your own words?"
