# Memory — 2026-05-01

## Session: GymForge Debug + Launch

### What We Did
Full debug session on GymForge — started from a 502 dead app, ended with all 7 roles green and zero errors.

### Root Cause: Stale Railway Postgres Migration Records
The Railway Postgres DB had migration records marked as "applied" but the actual tables were missing (DB was likely wiped/reset at some point). Django refused to re-create tables it thought already existed. This caused cascading 500s across all portals.

### Fixes Applied (all committed + pushed to GitHub)
1. **`gym/0001_initial.py`** — was completely missing from the repo. Django crashed on every boot. Wrote it manually from the `GymConfig` model definition.
2. **86 missing DB tables** — wrote `0002_ensure_tables.py` repair migrations for every app using `CREATE TABLE IF NOT EXISTS`. Then wrote a nuclear `core/0003_create_all_missing_tables.py` that uses Django's schema editor to auto-create every missing table.
3. **52 missing DB columns** — wrote `core/0004_fix_missing_columns.py` and `core/0005_add_missing_columns.py` using `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`.
4. **`request.tenant` remnant** — old multi-tenant code in member home view. Fixed to read from `GymConfig.get()`.
5. **`user.memberprofile` accessor** — OneToOneField has `related_name='member_profile'`. Fixed all views to use `user.member_profile`.
6. **Missing `_get_member_profile()` helper** — called in `members/views.py` but never defined. Added the function.
7. **Leads URL namespace** — moved leads include to top-level `config/urls.py` so `leads:pipeline` resolves correctly.
