### Architecture Patterns (what works across all apps)

**Standard app structure (all apps):**
- `app.py` monolith (Flask) — single file, all routes
- SQLite at `/data/` (Railway volume for persistence)
- `gunicorn app:app --bind 0.0.0.0:$PORT` start command
- `railway.json` for deploy config
- `/health` endpoint returning `{"status":"ok","db":"ok"}`
- WAL mode: `PRAGMA journal_mode=WAL` + `synchronous=NORMAL`
- Rate limiting: SQLite-based, 10 login attempts/min
- Session cookies: `httponly=True, samesite=Lax`

**Auth system (consistent across all apps):**
- SHA256 or bcrypt password hashing
- `login_required` decorator
- `admin_required` / `overseer_required` for admin routes
- `ADMIN_USER='admin'`, `ADMIN_PASS=os.environ.get('ADMIN_PASSWORD','admin123')`  ← NOTE: default is admin123 not admin1
- `session.permanent=True` + 8h lifetime
- `plan_required()` decorator for feature gating

**Multi-tenant pattern (Liberty Inventory + Consignment + others being upgraded):**
- Tenants = client stores under `/data/customers/<slug>/`
- `active_store_slug()` → checks `session['impersonating_slug']` or `session['store_slug']`
- `get_store_paths(slug)` → returns dict of paths for that tenant
- Overseer can impersonate any tenant

### Critical Lessons Learned (from research/ + sessions)

1. **Default admin password is `admin123` NOT `admin1`** — confirmed from app code. `admin1` only if `ADMIN_PASSWORD=admin1` set in Railway env vars
2. **Secret key persistence** — list-it-everywhere solved this correctly: persist SECRET_KEY to `/data/secret_key` file so sessions survive restarts. Other apps use env var fallback. If env var not set + redeploy → sessions invalidated
3. **`session=; Max-Age=0`** on login response = sessions being cleared = either wrong password OR secret key changed
4. **url_for() errors 500** — always double-check function name matches `url_for()` call. `product_view` vs `view_product` caused a 500
5. **Jinja `{% raw %}{% endraw %}`** — needed around CSS with `{}` braces inside `<style>` blocks in Flask templates. Removing causes 500.
6. **Railway volume** — all data at `/data/`. `start.sh` must `mkdir -p $DATA_DIR/uploads` etc before gunicorn starts
7. **CACHEBUST in Dockerfile** — bump `ARG CACHEBUST=<timestamp>` to force Railway to rebuild without cache
8. **OpenRouter vision models** — only use: `google/gemini-flash-1.5`, `google/gemini-2.0-flash-001`, `openai/gpt-4o`, `openai/gpt-4o-mini`, `anthropic/claude-3-haiku`, `anthropic/claude-3.5-sonnet`
9. **CSRF tokens** — the `_validate_csrf()` function exists in apps but is NOT automatically called. Must manually add `<input type="hidden" name="csrf_token" value="{{ csrf_token() }}">` to every POST form
10. **jay-portfolio branch is `master` not `main`** — always push to master for that repo
